# wuhan-timer.js 脚本解释文档

## 1. 脚本用途

`wuhan-timer.js` 是一个运行在 `*.edu-edu.com` 页面上的油猴脚本，用来自动完成网课视频的常见学习流程：

- 自动恢复播放
- 视频结束后自动跳转下一章
- 自动清理常见弹窗遮挡
- 提供悬浮控制面板（开关、倍速、音量、检测间隔）
- 在长时间检测不到视频时可自动刷新页面

脚本入口文件：`/Users/leonx/.openclaw/workspace/scripts/wuhan-timer.js`

---

## 2. 运行流程（整体架构）

脚本采用模块化对象组织，主流程如下：

1. 防重复初始化（同一页面只运行一个实例）
2. 读取本地配置（`localStorage`）
3. 页面就绪后创建控制面板
4. 启动章节检测 + DOM 变化监听
5. 启动定时器循环执行 `tick()`
6. 每次 `tick()` 执行：
   - 更新章节状态
   - 清理弹窗
   - 检测视频状态并执行自动播放/自动下一章等动作

## 3. 主要模块说明

### 3.1 CONFIG（固定参数）

用于定义脚本内部常量，例如：

- 默认检测间隔 `checkInterval`
- 章节检测间隔 `chapterDetectIntervalMs`
- 点击节流 `clickDelayMs`
- 视频结束阈值 `videoEndedThreshold`
- 无视频自动刷新阈值 `noVideoReloadMs`

这些是“程序级默认值”，最终运行时可被用户配置覆盖（例如检测间隔）。

### 3.2 Logger（日志模块）

统一输出日志前缀 `[武汉理工-增强版]`。

- `log`：仅在 `debug` 开启时输出
- `warn` / `error`：始终输出

便于在控制台快速筛选脚本日志。

### 3.3 Settings（配置持久化）

负责配置的读取与保存：

- 读取：`Settings.load()`
- 保存：`Settings.save(partial)`
- 存储键：`wuhan.timer.v11.settings`

默认配置包括：

- `autoPlay` 自动播放
- `autoNextChapter` 自动下一章
- `autoCloseDialogs` 弹窗清理
- `reloadWhenNoVideo` 无视频自动刷新
- `muted` 静音
- `volume` 音量
- `playbackRate` 倍速
- `checkInterval` 检测间隔

### 3.4 State（运行时状态）

保存脚本的运行时信息，例如：

- 当前章节列表与当前章节
- 是否正在处理点击（防止重复点击）
- 监控定时器 ID
- 无视频开始时间
- 最近处理过的视频标识（防止重复触发下一章）
- 是否顶层窗口、面板是否收起等 UI 状态

### 3.5 UI（悬浮面板）

负责注入样式、创建面板、绑定事件、同步状态。

特性：

- 仅在顶层窗口显示（避免 iframe 重复出现）
- 支持“收起/展开”
- 实时修改配置并持久化
- 支持手动“播放”和“下一章”

### 3.6 DialogHandler（弹窗清理）

定时尝试关闭常见弹窗容器（Element/Ant 等常见类名），降低遮挡导致的无法操作问题。

### 3.7 NextButtonNavigator（按钮兜底跳转）

当章节列表点击失败时，尝试按按钮文本匹配并点击：

- 下一章 / 下一节 / 下一个 / 继续学习 / next / continue

这是“章节列表策略”的兜底方案。

### 3.8 ChapterDetector（章节检测与切换）

职责：

- 检测章节节点（`.videoBox` 系列选择器）
- 识别当前章节（active 类 + URL 回退匹配）
- 计算下一章节
- 执行章节点击（链接点击 + 事件派发）

同时通过：

- `startPeriodicDetection()` 周期重试
- `startDOMObserver()` 监听 DOM 变化后重新检测

增强动态页面适配能力。

### 3.9 VideoDetector（视频检测与策略执行）

核心逻辑：

1. 获取 `video` 元素
2. 应用用户配置（倍速、静音、音量）
3. 判断播放状态：
   - 暂停：若开启自动播放则 `play()`
   - 结束：触发自动下一章（带去重）
   - 正常播放：按进度分段输出日志
4. 无视频：按配置可在超时后刷新页面

### 3.10 Controller（启动与主循环）

生命周期：

- `init()`：初始化设置、判断顶层窗口、进入 DOM 就绪流程
- `onDOMReady()`：创建 UI、延时启动
- `startUp()`：启动章节检测和监控
- `startMonitoring()`：开启主定时器
- `restartMonitoring()`：动态切换检测间隔
- `tick()`：单轮执行章节检测 + 弹窗清理 + 视频检测

## 4. 控制面板字段说明

- 自动播放：视频暂停时自动恢复
- 自动下一章：视频结束后自动切换
- 弹窗清理：自动尝试关闭阻挡弹窗
- 无视频自动刷新：长时间没检测到视频时刷新页面
- 调试日志：输出详细日志
- 静音：直接设置 `video.muted`
- 倍速：设置 `video.playbackRate`
- 音量：设置 `video.volume`（0~1）
- 检测间隔：控制主循环频率（1.5s / 3s / 5s）

---

## 5. 稳定性设计

脚本内做了多处保护：

- 防重复注入：全局标识位避免二次初始化
- 顶层窗口判断：避免 iframe 多面板
- 点击节流：`isProcessing` 防止连续误点击
- 视频结束去重：同一视频不会重复触发下一章
- 无视频超时策略：告警 + 可选自动刷新

## 6. 常见问题排查

### Q1：不自动下一章

建议依次检查：

1. 面板里“自动下一章”是否开启
2. 控制台是否有“视频已结束”日志
3. 当前页面是否存在下一章按钮文本（用于兜底）
4. 章节选择器是否命中（控制台会输出章节检测日志）

### Q2：看不到面板

可能原因：

- 当前不是顶层窗口（脚本刻意不在 iframe 渲染）
- 页面 CSS 强覆盖导致显示异常

### Q3：视频不自动播放

可能原因：

- 浏览器自动播放策略限制（可尝试先手动点击页面一次）
- 面板中“自动播放”关闭
- 页面并非标准 `video` 标签播放器
